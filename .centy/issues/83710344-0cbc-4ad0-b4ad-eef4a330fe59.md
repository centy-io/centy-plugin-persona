# Use centy-io/proto as git submodule instead of holding proto files locally

## Summary

This repo currently holds proto file(s) directly in the `proto/` directory. The canonical proto definitions live in the [centy-io/proto](https://github.com/centy-io/proto) repo.

## Task

Replace the local `proto/` directory with a git submodule pointing to `https://github.com/centy-io/proto.git`.

**Steps:**
1. Remove the existing `proto/` directory from the repo
2. Add the proto repo as a submodule: `git submodule add https://github.com/centy-io/proto.git proto`
3. Update any build scripts or tooling that references the proto files to work with the submodule path
4. Commit `.gitmodules` and the submodule reference

## Why

Centralising proto definitions in a single repo avoids drift and duplication across services. Any proto change is made once in `centy-io/proto` and all consumers pull the updated version via the submodule.
